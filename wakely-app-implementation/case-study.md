# Wakely App Implementation

## 1. Overview
- **Date: June 2026**
- **Technical Summary: Implementation of Wakely AOR and ACOS Modules**

## 2. Problem Statement
- I was responsible for developing the Wakely AOR (ACO Optimization Retrosight) and AOF (ACO Optimization FutureSight) modules within the application.
- The work required integration across Snowflake, SQL Server, React, and .NET within an existing MVC architecture.
- The project also demanded close coordination with both internal stakeholders and the client to ensure the new modules met functional and business expectations.

## 3. Approach
- The implementation required cross-team communication with the client and Agile-style development, as I regularly worked with the Product team to refine and improve the modules before deployment.
- I also applied agentic AI in a way that supported development while maintaining appropriate coding standards and security practices, which was especially important given the sensitivity of the application data.

## 4. Execution Summary
- I used agentic AI to help transition the work from an online mock-up to a practical application skeleton while staying aligned with existing UI patterns and coding conventions.
- I worked with the data team to obtain the necessary information from Snowflake and used SQL Server to store results for faster retrieval and access.
- I then implemented the React-based front end to present the data, including lazy loading to manage large volumes of records and to support user permissions for viewing and creating data as well as data shared by other users.

## 5. Results
- The outcome was two functional applications that enabled users to review historical TIN (Taxpayer Identification Number) and CCN (CMS Certification Number) expenditure data and compare that history with projected future performance under different scenarios.

## 6. Key Learnings
- I developed a stronger understanding of how to use agentic AI safely and effectively in application development.
- I also improved my knowledge of Entity Framework connections to both SQL Server and Snowflake and learned how to use each platform efficiently for data retrieval.
- The project strengthened my ability to integrate React-based front ends with backend APIs for retrieval and display, with a focus on creating a smoother user experience.

## 7. Impact
- The new feature successfully presented historical and predictive results in a way that helped users compare performance trends while respecting access controls and security requirements.
- By making historical and predictive data more accessible, the feature supports better-informed business decisions for administrators.
