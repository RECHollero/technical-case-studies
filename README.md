## Case Studies Table of Contents

### 1. [Supervisor Grid Performance Improvements](./supervisor-grid-performance-improvement/case-study.md)
* **The Problem:** SlickGrid implementation and custom filtering caused significant performance improvements when a lot of patient data was loaded into the supervisor grid.
* **The Solution:** Custom group pagination, filtering, and searching, as well as lazy loading allowed for significant performance improvements to the grid.
* **Core Tech:** TypeScript, ASP.NET, SQL Server, Entity Framework.
* **Business Impact:** Improved grid interaction performance from tens of seconds to milliseconds.

### 2. [Wakely App Implementation](./wakely-app-implementation/case-study.md)
* **The Problem:** The Wakely application required the development of AOR and AOF modules that could integrate historical and predictive performance data across Snowflake, SQL Server, React, and .NET within an existing MVC architecture.
* **The Solution:** Implemented the new modules through coordinated cross-team development, agentic AI-assisted migration support, and React-based UI work that incorporated lazy loading and permission-aware data access.
* **Core Tech:** React, ASP.NET, SQL Server, Snowflake, Entity Framework.
* **Business Impact:** Delivered functional modules that enabled users to compare historical and projected provider performance data while supporting secure, permission-aware access.
